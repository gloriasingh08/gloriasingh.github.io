Assignment Creation

The application's main objective is to create and manage assignments created by authenticated users and posting assignments with input validation. The web application will load account information from a well-known location. The application should load these accounts at startup and create users based on the information provided in the CSV file. The web applicationâ€™s database must be the RDS instance launched by Pulumi when running on the EC2 instance. The created assignments will be stored in AWS RDS with the database engine: MariaDB The user will provide a basic authentication token when making an API call to any authenticated endpoint. Added a GitHub Actions workflow to run the application tests for each pull request raised. A pull request can only be merged if the workflow executes successfully. Added Status Checks GitHub branch protection to prevent users from merging a pull request when the GitHub Actions workflow runs. The application has infrastructure as a code section where networking resources such as Virtual Private Cloud (VPC), Internet Gateway, Route Table, and Routes are defined, using Pulumi for infrastructure setup and tear down. The application has to build custom machine images(Amazon Machine Image) using Packer that can be used to create virtual machines in the cloud using Pulumi for Infrastructure as Code. The AMI should include everything needed to run the application including the shell script for running all the dependencies. The EC2 template for the load balancing group should be launched with user data. Doing regular health checks by Integration testing whenever there is a pull request. Route53 will be updated by Pulumi, adding different records like CNAME, TXT, and NS to the Route53 zone so that the domain points to the EC2 auto-balancing group where the web application is accessible. SSL certificates were imported for the application for secure browsing, as SSL/TLS certificates are a fundamental component of the HTTPS protocol, ensuring secure and encrypted communication between web browsers and servers. When a user is posting any assignment, the assignment status is sent to the user after successful submission and also if the URL is invalid. For sending email services the application uses Amazon SES services which will be triggered when there is a post REST API and the assignment is posted to an SNS topic in AWS, which in turn will trigger AWS lambda, and then an email will be sent using SES and also the email deliveries are stored inside dynamo db to keep track of all the emails. This part of the application uses serverless computing. The submission URLs are saved inside Google Bucket for versioning, as the same user can post many versions of the same assignment.

function sendSnsMessage(username, submissionUrl,assignmentId, status) {
    const snsTopicArn = process.env.snsTopicArn;
    if (!snsTopicArn) {
        logger.error('SNS Topic ARN is not defined in environment variables');
        return;
    }
    
    const message = {
        user: username,
        submissionUrl: submissionUrl,
        assignmentId:assignmentId,
        status: status
    };
    
    const params = {
        Message: JSON.stringify(message),
        TopicArn: snsTopicArn
    };
    
    sns.publish(params, (err, data) => {
        if (err) {
            logger.error('Error publishing to SNS:', err);
        } else {
            logger.info('Successfully published status to SNS:', data);
        }
    });
}
